How to detect hacker groups?

Before we talk about hacking groups and the attacks they carry out, let's discuss how we determine if an attack came from a specific hacking group. There are a few key attributes that need to be examined before we can make that determination:

  1. Malware and Tools Used
  1. Examining Tactics, Techniques, and Procedures
  1. Investigating Infrastructure
  1. Examining Targets
  1. Public and Private Reports

Let's start from the beginning.

1) Malware and Tools Used

Imagine a company called “Please Don't Hack Me.” This company has been breached (meaning hacked). The necessary personnel have isolated the malware, and the system is now up and running again. It’s time to analyze the malware, and they need to document the situation so that if other companies get attacked, they'll know what to do. The code or tools used during an attack can often be similar to previous attacks. In many cases, code from a previous attack can be modified and reused. Some groups are also known for using unique techniques every time. The next steps help determine the identity of the attackers.

2) Examining Tactics, Techniques, and Procedures

Even though different (or unique) malware might be used to gain access to a system, certain things can still be identified. Even if the code is different, how it was delivered, the mission of that malware, and how the supply chain of attacks occurred can all be used to determine the hacking group responsible.

Spoiler: Some groups publicly claim responsibility for an attack to gain more recognition in the media.

3) Investigating Infrastructure

There is a term called "C2," which stands for Command and Control. Once malware is in the wild, it has to do something—often it disrupts the system, shuts down infrastructure, or stops the ongoing work of the company. In some cases, it steals information. That information has to be sent somewhere (meaning back to where the attack originated), which can be tracked down. Analysts check if these servers have been used previously by other hacker groups.

4) Examining Targets

Let’s say the “Please Don’t Hack Me” company handles important financial deals in a country called “Never Been Hacked Before.” Last night, this country had some issues with another country called “Going to Hack You.” The next day, the “Please Don’t Hack Me” company gets hacked, and their important financial infrastructure is now down. Guess who might have done it?

5) Public and Private Reports

As a last resort, all the steps above are documented. Some people might skip doing all of them and simply search the internet for similar effects that have occurred in other systems. If they’re lucky, they can find a good report on the situation and will be able to stop the attack before it spreads and affects more infrastructure.
